1. 테스트 환경
- echo TPS를 50만으로 맞추고 연결 테스트를 함.
- 원래 150만 TPS로 맞추고 연결 끊김 없는 부하 테스트를 먼저 해야 했으나 더미 클라이언트를 돌리는 노트북 성능이 좋지 않아 CPU 사용률이 계속 100% 먹으면서 제대로 된 테스트가 불가능 하여 연결 끊김 테스트만 진행 함.

2. connectFail
- 더미에서 connect() 호출 했는데 실패한 것임. 어차피 Connect Total에는 포함되지 않으니 문제 없음.

3. 게임 서버 잔존 세션 및 유저
- 더미 자체적으로 연결을 끊어서 더미에서 보이는 Connect / Login 수치를 0으로 만드는 기능이 없음.
- 그래서 더미를 그냥 끊어 버리면 RST를 한방에 게임 서버로 보내기 때문에 유실의 가능성이 있음. 
  그래서 게임 서버 쪽에 유저와 세션이 남아 있게 됨. cmd 창에서 nestat -ano로 보면 게임 서버의 소켓들이 보이는데 established 상태로 게임 서버에서 보이는 잔존 세션 갯수와 동일함.
